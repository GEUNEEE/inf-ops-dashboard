# 인플루언서 브리핑 자동화 에이전트 — 통합 설계서

> 작성일: 2026-05-06  
> 인터뷰 보완: 2026-05-06  
> 목적: Claude Code에서 구체적인 구현 시 참조할 계획서

---

## 1. 작업 컨텍스트

### 배경 및 목적
인플루언서 관리 엑셀과 구글 캘린더를 읽어, 매일 아침 담당자가 오늘 하루와 향후 7일의 인플루언서 일정 및 개인 일정을 한눈에 파악할 수 있도록 브리핑 문서를 자동 생성한다.

### 범위
- 인플루언서 일정: 엑셀 파일 기반, 오늘~오늘+7일 해당 항목
- 개인 일정: 구글 캘린더 기반, 오늘 당일 전체 + 오늘~오늘+7일

### 입력
| 입력 | 형태 | 위치 |
|------|------|------|
| 인플루언서 관리 엑셀 | `.xlsx` | `C:\Users\user\비서\스케줄\` (최신 파일 자동 감지) |
| 구글 캘린더 일정 | API (MCP) | Google Calendar MCP (기연결) |

### 출력
| 출력 | 형태 | 위치 |
|------|------|------|
| 브리핑 문서 | `.md` | `C:\Users\user\비서\브리핑결과\YYYY-MM-DD_브리핑.md` |
| 중간 파싱 결과 | `.json` | `C:\Users\user\비서\output\YYYY-MM-DD_parsed.json` |

### 제약조건
- 브리핑은 매일 아침 6시에 수동 또는 요청 시 즉시 실행 (자동 스케줄 미포함)
- 텔레그램 전송은 현재 범위 외 (추후 확장)
- 엑셀 구조: 열(Column) 기준으로 인플루언서 1명, 행(Row) 기준으로 항목 나열하는 전치(Transpose) 구조

### 용어 정의
| 용어 | 정의 |
|------|------|
| 7일치 | 오늘(실행일) ~ 오늘+7일, 요일 고정 없음 |
| 이번 주 체크 항목 | Row 28~34에 해당하는 날짜 컬럼 |
| 진행상태 | Row 10에 기록된 인플루언서별 현재 단계 |
| 조치 필요 항목 | Row 28~34 중 날짜가 오늘보다 과거인 항목 (무너진 마감) |

## 2. 워크플로우 정의

### 전체 흐름

```
[실행 트리거: '브리핑' 또는 '브리핑 실행해줘' 키워드]
        │
        ▼
  ┌─────────────────────────────────┐
  │  STEP 0. PowerShell 트리거       │  run_briefing.ps1
  │                                 │
  │  - venv 활성화                   │
  │  - extract.py 실행              │
  └──────────────┬──────────────────┘
                 │ (extract.py 완료 후 Claude Code에 트리거)
                 ▼
  ┌─────────────────────────────────┐
  │  STEP 1. 엑셀 파싱 (스크립트)    │  extract.py
  │                                 │
  │  - 최신 .xlsx 파일 감지          │
  │  - 인플루언서별 데이터 파싱       │
  │  - 오늘~오늘+7일 날짜 필터링     │
  │  - D-day 계산 (days_until)      │
  │  - overdue 항목 집계            │
  │  - JSON 파일로 저장 및 stdout 출력│
  └──────────────┬──────────────────┘
                 │ JSON
                 ▼
  ┌─────────────────────────────────┐
  │  STEP 2. 캘린더 조회 (LLM)       │  Claude Code
  │                                 │
  │  - Google Calendar MCP 호출     │
  │  - 오늘 당일 전체 일정 조회      │
  │  - 내일(+1일)~오늘+7일 일정 조회 │
  │  - 모든 캘린더 포함             │
  └──────────────┬──────────────────┘
                 │ 캘린더 데이터
                 ▼
  ┌─────────────────────────────────┐
  │  STEP 3. 브리핑 생성 (LLM)       │  Claude Code
  │                                 │
  │  - JSON + 캘린더 데이터 통합     │
  │  - 비고 1-2줄 LLM 요약          │
  │  - 섹션 순서에 맞게 MD 생성      │
  │  - 브리핑결과 폴더에 저장        │
  └─────────────────────────────────┘
```

### 단계별 상세 정의

#### STEP 0. PowerShell 트리거

| 항목 | 내용 |
|------|------|
| 담당 | `run_briefing.ps1` |
| 역할 | venv 활성화 + extract.py 실행만 담당 |
| 이후 | 사용자가 Claude Code에서 '브리핑 실행해줘' 입력 → STEP 2~3 진행 |

#### STEP 1. 엑셀 파싱 (스크립트 처리)

| 항목 | 내용 |
|------|------|
| 담당 | `extract.py` (스크립트) |
| 입력 | 엑셀 폴더 내 최신 `.xlsx` |
| 출력 | `output/YYYY-MM-DD_parsed.json` + stdout |
| 성공 기준 | JSON에 `influencers` 배열 존재, `date_range` 필드 포함, 인플루언서 1명 이상 |
| 검증 방법 | 스키마 검증 (필수 필드 존재 여부) |
| 실패 시 처리 | 엑셀 파일 없음 → 에러 메시지 출력 후 중단 / 파싱 오류 → 해당 인플루언서 스킵 + 로그 기록 |

#### STEP 2. 캘린더 조회 (LLM 처리)

| 항목 | 내용 |
|------|------|
| 담당 | Claude Code (Google Calendar MCP) |
| 입력 | 실행일 기준 날짜 범위 |
| 출력 | 오늘 전체 일정 + 내일~+7일 일정 (인라인) |
| 캘린더 범위 | 모든 캘린더 포함 (개인·업무 등 구분 없음) |
| 성공 기준 | MCP 응답 정상 수신, 날짜 범위 내 이벤트 목록 반환 |
| 실패 시 처리 | MCP 응답 실패 → 1회 재시도 → 실패 시 캘린더 섹션 "조회 실패" 표기 후 계속 진행 |

#### STEP 3. 브리핑 생성 (LLM 처리)

| 항목 | 내용 |
|------|------|
| 담당 | Claude Code |
| 입력 | STEP 1 JSON + STEP 2 캘린더 데이터 |
| 출력 | `브리핑결과/YYYY-MM-DD_브리핑.md` |
| 비고 처리 | JSON `note` 원문을 LLM이 1-2줄로 요약하여 삽입. `note`가 비어있으면 생략 |
| 성공 기준 | 필수 섹션 모두 포함, 파일 정상 저장 |
| 검증 방법 | LLM 자기 검증 (섹션 누락 여부 확인 후 재생성) |
| 실패 시 처리 | 섹션 누락 → 자동 재생성 (최대 1회) |

### 브리핑 섹션 순서

```
1. 👥 인플루언서 일정    (오늘~오늘+7일, 엑셀 기반)
2. ⚠️ 조치 필요 항목    (Row 28~34 중 오늘 이전 날짜, 엑셀 기반) ← 없으면 생략
3. 🗓️ 주간 캘린더 일정  (내일~오늘+7일, 구글 캘린더)
4. 📅 오늘 일정          (당일 전체, 구글 캘린더)
```

### 스킬 목록

| 스킬명 | 역할 | 트리거 조건 |
|--------|------|------------|
| `excel-parser` | 엑셀 파싱 → JSON 출력 | 브리핑 실행 시 STEP 1에서 항상 호출 |

#### `excel-parser` 스킬 상세

- **스크립트**: `extract.py`
- **입력**: 엑셀 폴더 경로, 실행일
- **출력**: `output/YYYY-MM-DD_parsed.json`
- **출력 JSON 구조**:

```json
{
  "generated_at": "2026-05-06 06:00",
  "date_range": { "start": "2026-05-06", "end": "2026-05-13" },
  "influencers": [
    {
      "name": "홍길동TV",
      "status": "1차 체험진행",
      "status_emoji": "🔵",
      "note": "제품 수령 확인 완료. 콘텐츠 방향 협의 중.",
      "upcoming": [
        { "item": "체험 종료",          "date": "2026-05-08", "days_until": 2 },
        { "item": "중간 관리일(7일차)", "date": "2026-05-10", "days_until": 4 }
      ],
      "overdue": [
        { "item": "1차 콘텐츠 미팅", "date": "2026-05-03", "days_overdue": 3 }
      ]
    },
    {
      "name": "이영희Vlog",
      "status": "기타",
      "status_emoji": "❌",
      "note": "",
      "upcoming": [],
      "overdue": []
    }
  ]
}
```

> `upcoming`이 비어있으면 해당 인플루언서는 브리핑에서 완전히 숨김  
> `note`가 비어있으면 비고 줄 생략  
> `overdue`가 비어있으면 조치 필요 섹션에서 제외

### 주요 산출물 파일 형식

| 파일 | 형식 | 설명 |
|------|------|------|
| `YYYY-MM-DD_parsed.json` | JSON | 엑셀 파싱 중간 결과 |
| `YYYY-MM-DD_브리핑.md` | Markdown | 최종 브리핑 문서 |

## 4. 도메인 지식

### 엑셀 파일 감지 규칙

- **파일 위치**: `C:\Users\user\비서\스케줄\`
- **파일명 패턴**: `인플루언서 관리_YYMMDD.xlsx` (예: `인플루언서 관리_260504.xlsx`)
- **최신 파일 선택**: 파일명에서 YYMMDD 숫자가 가장 큰 파일 선택
- **날짜 없는 파일 무시**: 패턴에 맞지 않는 파일은 건너뜀
- **중복 날짜**: 같은 YYMMDD가 여러 개면 파일명 사전순으로 마지막 것 선택

### 엑셀 구조 (전치 구조)

- **시트명**: `인플루언서관리`
- **열(Column)** 기준: 인플루언서 1명 = 열 1개 (B열부터 시작)
- **행(Row)** 기준: 항목 나열
- **열 스캔 종료 기준**: Row 11(유튜버명)이 비어있는 열에서 스캔 중단

| Row | 항목 | 비고 |
|-----|------|------|
| 10 | 진행상태 | 자유 텍스트 — strip + 공백 제거 후 퍼지 매치 |
| 11 | 유튜버명 | **브리핑 표시명**, 비어있으면 해당 열 스킵 |
| 12 | 이름 | 미사용 (브리핑에 표시 안 함) |
| 20 | 비고 | 원문 → JSON `note` 필드 → LLM 1-2줄 요약 |
| 24 | 체험 수락일 | |
| 25 | 제품 발송일 | |
| 26 | 수령 확인일 | |
| 27 | 섭취 시작일 | |
| 28 | 체험 종료 | ⭐ 7일치 체크 / ⚠️ overdue 체크 |
| 29 | 중간 관리일(7일차) | ⭐ 7일치 체크 / ⚠️ overdue 체크 |
| 30 | 1차 콘텐츠 미팅 | ⭐ 7일치 체크 / ⚠️ overdue 체크 |
| 31 | 1차 광고 | ⭐ 7일치 체크 / ⚠️ overdue 체크 |
| 32 | 2차 체험 | ⭐ 7일치 체크 / ⚠️ overdue 체크 |
| 33 | 2차 콘텐츠 미팅 | ⭐ 7일치 체크 / ⚠️ overdue 체크 |
| 34 | 2차 광고 | ⭐ 7일치 체크 / ⚠️ overdue 체크 |

> **빈 날짜 셀 처리**: Row 28~34 날짜가 비어있으면 스킵 (upcoming/overdue 모두 제외)  
> **Row 24~27**: overdue 체크 대상 아님 (인수 단계 항목)

### 진행상태 이모지 매핑

| 상태값 | 이모지 | 매핑 규칙 |
|--------|--------|----------|
| 1차 체험진행 | 🔵 | strip + 공백 제거 후 매칭 |
| 1차 광고예정 | 🟡 | strip + 공백 제거 후 매칭 |
| 1차 광고완료 | ✅ | strip + 공백 제거 후 매칭 |
| 2차 체험진행 | 🟣 | strip + 공백 제거 후 매칭 |
| 2차 광고예정 | 🟠 | strip + 공백 제거 후 매칭 |
| 2차 광고완료 | ✅✅ | strip + 공백 제거 후 매칭 |
| 기타 | ❌ | strip + 공백 제거 후 매칭 |
| (매핑 실패) | ❓ | 원문 그대로 표시 |

### 인플루언서 정렬 규칙

- **기준**: `upcoming[0].date` (가장 이른 예정 일정) 기준 오름차순
- **upcoming 없는 인플루언서**: 브리핑 인플루언서 섹션에서 완전히 숨김

### 브리핑 출력 형식 예시

