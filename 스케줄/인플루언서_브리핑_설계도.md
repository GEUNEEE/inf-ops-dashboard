# 📋 인플루언서 주간 브리핑 자동화 — 설계도

> 작성일: 2026-05-06  
> 목적: 인플루언서 엑셀 + 구글 캘린더를 읽어 오늘부터 7일치 일정을 매일 아침 브리핑

---

## 1. 워크플로우 분리 원칙

| 구분 | 담당 | 이유 |
|------|------|------|
| **스크립트 워크플로우** | 엑셀 읽기 / 데이터 파싱 / 날짜 필터링 / JSON 출력 | 정형 데이터 처리는 스크립트가 정확하고 빠름 |
| **LLM 워크플로우** | 구글 캘린더 조회 / JSON 수신 / 브리핑 문장 생성 / MD 파일 저장 | 자연어 정리·요약 + 캘린더 MCP 활용은 LLM이 유리 |

---

## 2. 전체 흐름

```
[매일 오전 6:00 - Windows 작업 스케줄러 자동 실행]
        │
        ▼
  ┌──────────────────────────────────┐
  │   STEP 1: 스크립트 워크플로우      │  extract.py
  │                                  │
  │  - 엑셀 폴더에서 최신 파일 감지    │
  │  - 인플루언서별 데이터 파싱        │
  │  - 오늘 ~ 오늘+7일 해당 항목 필터  │
  │  - 결과를 JSON으로 stdout 출력    │
  └──────────────┬───────────────────┘
                 │ JSON (인플루언서 일정)
                 ▼
  ┌──────────────────────────────────┐
  │   STEP 2: LLM 워크플로우          │  Claude Code
  │                                  │
  │  - extract.py JSON 수신          │
  │  - 구글 캘린더 MCP 조회           │
  │    · 오늘 일정                   │
  │    · 오늘~오늘+7일 일정           │
  │  - 두 데이터 합쳐 브리핑 생성      │
  │  - MD 파일로 저장                │
  └──────────────────────────────────┘
```

---

## 3. 날짜 기준

| 순서 | 항목 | 기준 |
|------|------|------|
| 1 | 인플루언서 일정 | 오늘 ~ 오늘+7일 |
| 2 | 구글 캘린더 (주간 일정) | 오늘 ~ 오늘+7일 |
| 3 | 구글 캘린더 (오늘 일정) | 실행일 당일 전체 |

> 예) 5월 6일(화) 실행 시 → 5/6(화) ~ 5/13(화) 기준

---

## 4. STEP 1 — 스크립트 워크플로우 (extract.py)

### 역할
- 엑셀 파일 탐색 및 읽기
- 인플루언서 데이터 파싱
- 오늘 ~ 오늘+7일 날짜 필터링
- 결과를 JSON으로 stdout 출력

### 출력 JSON 구조

```json
{
  "generated_at": "2026-05-06 06:00",
  "date_range": { "start": "2026-05-06", "end": "2026-05-13" },
  "influencers": [
    {
      "name": "홍길동TV",
      "status": "1차 체험진행",
      "upcoming": [
        { "item": "체험 종료",          "date": "2026-05-08" },
        { "item": "중간 관리일(7일차)", "date": "2026-05-10" }
      ]
    },
    {
      "name": "김철수Tube",
      "status": "1차 광고예정",
      "upcoming": [
        { "item": "1차 콘텐츠 미팅", "date": "2026-05-07" },
        { "item": "1차 광고",        "date": "2026-05-09" }
      ]
    },
    {
      "name": "이영희Vlog",
      "status": "기타",
      "upcoming": []
    }
  ]
}
```

---

## 5. STEP 2 — LLM 워크플로우 (Claude Code)

### 역할
- extract.py JSON 수신 (인플루언서 일정)
- 구글 캘린더 MCP로 오늘 일정 + 7일치 일정 조회
- 두 소스를 합쳐 브리핑 MD 생성 및 저장

### 구글 캘린더 조회 항목

| 조회 | 범위 |
|------|------|
| 오늘 일정 | 실행일 00:00 ~ 23:59 |
| 주간 일정 | 실행일 ~ 실행일+7일 |

### 브리핑 출력 형식 (MD 파일 내용)

```markdown
# 📋 일일 브리핑
🕕 2026-05-06 (화) 06:00 생성
📆 일정 범위: 5/6(화) ~ 5/13(화)

---

## 👥 인플루언서 일정 (5/6 ~ 5/13)

### 👤 홍길동TV
🔵 진행상태: 1차 체험진행
  • 5/8(목) 체험 종료
  • 5/10(토) 중간 관리일(7일차)

### 👤 김철수Tube
🟡 진행상태: 1차 광고예정
  • 5/7(수) 1차 콘텐츠 미팅
  • 5/9(금) 1차 광고

✅ 일정 있는 인플루언서: 2명  ⚠️ 일정 없는 인플루언서: 1명

---

## 🗓️ 주간 캘린더 일정 (5/6 ~ 5/13)
- 5/7(수) 13:00 인플루언서 계약 미팅
- 5/9(금) 광고 소재 마감
- 5/11(일) 제품 발송 마감

---

## 📅 오늘 일정 (5/6)
- 10:00 팀 미팅
- 14:00 거래처 통화
- 18:00 콘텐츠 검수
```

## 6. 엑셀 구조

### 전치(Transpose) 구조
- **열(Column)** 기준: 인플루언서 1명 = 열 1개
- **행(Row)** 기준: 항목 나열

### 행 인덱스 정의

| Row | 항목 | 비고 |
|-----|------|------|
| 10 | 진행상태 | 아래 상태값 중 하나 |
| 11 | 유튜버명 | |
| 12 | 이름 | |
| 24 | 체험 수락일 | |
| 25 | 제품 발송일 | |
| 26 | 수령 확인일 | |
| 27 | 섭취 시작일 | |
| 28 | **체험 종료** | ⭐ 7일치 체크 |
| 29 | **중간 관리일 (7일차)** | ⭐ 7일치 체크 |
| 30 | **1차 콘텐츠 미팅** | ⭐ 7일치 체크 |
| 31 | **1차 광고** | ⭐ 7일치 체크 |
| 32 | **2차 체험** | ⭐ 7일치 체크 |
| 33 | **2차 콘텐츠 미팅** | ⭐ 7일치 체크 |
| 34 | **2차 광고** | ⭐ 7일치 체크 |

## 8. 파일 구성

```
C:\Users\user\비서\
│
├── 스케줄\                          ← 엑셀 파일 보관 폴더
│   └── (최신 .xlsx 파일 자동 감지)
│
├── 브리핑결과\                       ← MD 출력 폴더 (없으면 자동 생성)
│   └── YYYY-MM-DD_브리핑.md          ← 실행일 기준 파일명 자동 생성
│
└── extract.py                       ← 스크립트 워크플로우 (엑셀 파싱 → JSON 출력)
```
